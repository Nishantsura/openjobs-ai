# Codex Model Usage Guide

## Use Codex 5.3 for

- Architecture-critical decisions
- Complex automation logic (LinkedIn adapters, mutation/race handling)
- Security-critical code paths
- Prompt design and schema hardening
- Debugging flaky and non-deterministic failures

## Use Codex 5.2 for

- Standard endpoint scaffolding
- Medium-complexity refactors
- Test generation and expansion
- Platform adapters after first stable pattern exists

## Use Codex 5.1 for

- Documentation updates
- Repetitive small edits
- Copy/text/UI wording updates
- Straightforward config touch-ups
